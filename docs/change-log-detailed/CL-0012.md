# CL-0012: Enforce tenant header at gateway

## Summary
This change adds a gateway filter enforcing tenant headers for protected routes and documents the header requirement.

## Motivation
Tenant header enforcement is required to maintain multi-tenant isolation at the edge.

## Changes
- Added gateway global filter to require `x-tenant-id` for non-public routes.
- Documented the tenant header requirement in the system architecture.

## Verification
- Not run (filter addition only).

## Risks
- Requests without tenant headers will now receive 400 responses.

## Rollback
- Remove the tenant header filter and documentation entry.

## Notes
- Consider validating tenant ID format in a future iteration.

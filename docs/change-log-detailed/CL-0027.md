# CL-0027: Add account provisioning APIs

## Summary
This change adds account provisioning endpoints with basic validation and uniqueness checks.

## Motivation
Tenant administrators need a way to provision accounts in the identity service.

## Changes
- Added account controller and request/response DTOs.
- Added repository helpers for tenant-scoped uniqueness checks.
- Secured account endpoints under admin security config.
- Updated REST API documentation.

## Verification
- mvn -q -DskipTests=false verify (pass).

## Risks
- Validation is minimal and may require stronger constraints.

## Rollback
- Remove account controller, DTOs, and repository changes.

## Notes
